# Security Policy

## Supported Versions

We take the security of the Tripdhara platform very seriously. Currently, only the latest major release is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover any security vulnerability in the Tripdhara platform, please do **not** report it through public GitHub issues. 

Instead, please report it privately to our security team. We appreciate your efforts to responsibly disclose your findings.

### How to Report:
- **Email:** security@tripdhara.com (Please use a descriptive subject line like "Security Vulnerability Report")
- **WhatsApp Support:** Reach out directly via our official WhatsApp widget on the platform.

### What to include in your report:
- A clear description of the vulnerability and its impact.
- Steps to reproduce the issue.
- Any relevant logs, screenshots, or code snippets.

### Our Response Process:
1. We will acknowledge receipt of your vulnerability report within 48 hours.
2. We will investigate the issue and determine if it is a valid vulnerability.
3. If confirmed, we will work on a patch and notify you of our progress.
4. Once resolved, we will publish a security update and mention your responsible disclosure (if you consent).

Thank you for helping keep Tripdhara safe for our travelers!
